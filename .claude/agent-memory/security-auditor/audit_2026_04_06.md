Audit date: 2026-04-06. Scope: full (secrets, auth, API, database, infra, dependencies).

Risk Score: 7/10 at time of audit.

**Critical findings:**
1. Live credentials in backend/.env (OpenAI key, Gmail app password, Google OAuth secret, DB password) — file exists on disk but not committed to git.
2. JWT tokens exposed in URL query params after Google OAuth redirect (/auth/callback?access_token=...&refresh_token=...).
3. CORS set to allow_origins=["*"] with no production environment gate — main.py line 33.
4. Next.js 16.1.6 has CVE for HTTP request smuggling (GHSA-ggv3-7p47-pfv8) and CSRF Server Actions bypass (GHSA-mq59-m269-xvcx).

**High findings:**
1. OAuth missing state parameter — CSRF attack possible on /api/v1/auth/google (auth.py:139).
2. Widget session_id accepted from client without validation — session hijacking (chat.py:59).
3. Email verification not enforced before agent creation — business rule violated (agent_service.py has no is_verified check).
4. Plan limit hardcoded to FREE_PLAN_AGENT_LIMIT=3 (wrong; should be 1 for free plan per business rules, and ignores subscription table).
5. DEV reset token logged in plaintext at WARNING level (auth_service.py:311-314).

**Medium findings:**
1. API docs (Swagger/ReDoc/OpenAPI JSON) exposed in production — no is_production gate in main.py.
2. ChatRequest.message has no max_length — unbounded LLM cost/DoS.
3. AddTextRequest.content/title have no max_length validators.
4. AddUrlRequest.url has no schema-level validation (comment says "validated at runtime").
5. document_parser.py dispatches by file extension only — no MIME type validation.
6. No rate limiting implemented on any endpoint (Phase 8 not yet built, but authenticated endpoints also have no limiting).
7. Qdrant exposed on host ports 6333/6334 in docker-compose.yml with no auth.

**Info findings:**
1. asyncio.get_event_loop() deprecated in Python 3.10+ (email_service.py).
2. flatted npm package has prototype pollution CVE (GHSA-rf6f-7fwh-wjgh), high severity.
3. brace-expansion, picomatch npm CVEs (moderate/high) in dev dependencies.
4. Redis port 6379 exposed on host in docker-compose.yml (dev only, but risky).

**Why:** Preserved to allow comparison against future audits and track remediation.
**How to apply:** On next audit, check each finding to see which were remediated.
